1. Сведения о программе

Название: Веб-приложение по учету сотрудников предприятия
Заказчик: Ресторан Своя Компания
Разработчик: Тимофеев О.И., группа 285, УрТИСИ СибГУТИ

Программный продукт предназначен для автоматизации кадрового учета и реализован через:

BACKEND: ASP.NET Core
FRONTEND: HTML, CSS, JavaScript
База данных: PostgreSQL

2. Требования к программным и техническим средствам

2.1 Технические требования
Для стабильной работы серверной части приложения рекомендуется использовать сервер или ПК со следующими минимальными характеристиками:

процессор: 1-2 ядра x64 архитектура
оперативная память: не менее 1 ГБ для базовой работы
свободное место на диске: от 10 ГБ SSD предпочтительнее
сеть: скорость интернет-соединения от 2 Мбит/с для удаленного доступа

2.2 Программные требования
Для развертывания и запуска программы требуется следующее программное обеспечение:

операционная система: Linux Ubuntu/Debian или Windows Server поддерживается Windows 10/11 для разработки
NET SDK Runtime версия, используемая в проекте, см файл проекта
PostgreSQL версии 14 или выше
pgAdmin 4 или любая другая удобная СУБД-админка, например DBeaver
современный веб-браузер: Google Chrome, Яндекс Браузер, Microsoft Edge для доступа к интерфейсу

3. Работа с приложением

3.1 База данных

Скрипты для создания структуры базы данных CREATE TABLE и наполнения тестовыми данными INSERT находятся в проекте В документации листинг 21 приведен полный SQL-код для создания всех таблиц, индексов и связей

Таблицы: Employees, Departments, Positions, SystemUsers, Vacations, AuditLogs и другие
Кодировка данных: при создании БД рекомендуется использовать UTF-8


3.2 Настройка конфигурации приложения

Для корректной работы приложения необходимо настроить подключение к базе данных Файл конфигурации не выкладывается в публичный репозиторий в целях безопасности

В файле appsettingsjson или аналогичном файле конфигурации необходимо настроить строку подключения:

DefaultConnection: строка подключения к PostgreSQL в формате Host=localhost;Port=5432;Database=EmployeeAccounting;Username=your_user;Password=your_password

3.3 Первоначальный запуск

Убедитесь, что все необходимые NuGet пакеты установлены:

dotnet add package Microsoft.EntityFrameworkCore
dotnet add package Npgsql.EntityFrameworkCore.PostgreSQL
dotnet add package Microsoft.EntityFrameworkCore.Design

Остальные пакеты добавляются по необходимости, в зависимости от структуры проекта

В каталоге с проектом где лежит файл csproj выполните команды:

dotnet restore
dotnet build
dotnet run

После успешного запуска в консоли появится адрес обычно https://localhost:5001 или http://localhost:5000, по которому будет доступно веб-приложение

3.4 Проверка запуска
После старта необходимо убедиться что:

сервер запускается без ошибок в консоли
главная страница приложения открывается в браузере
поиск сотрудников работает
операции добавления редактирования и удаления проходят успешно
есть доступ к админ-панели журналы действий и входов

4. Структура проекта backend

Проект выполнен на базе ASP.NET Core:

Data/ содержит контекст базы данных, где объявлены таблицы и связи между ними
Models/ хранятся модели соответствующие таблицам базы данных Employees, Departments, Positions и другие
Controllers/ обработчики HTTP-запросов API-контроллеры с CRUD-операциями
appsettingsjson файл конфигурации включающий строку подключения к БД

5. Структура проекта frontend

HTML-страницы основная страница приложения главный экран с таблицей сотрудников и фильтрами
site.css или встроенные стили оформление интерфейса
JavaScript app.js или встроенный код клиентская логика: асинхронные запросы к API, рендеринг таблиц, обработка действий пользователя

6. Возможные ошибки

6.1 Приложение не запускается ошибки при dotnet run
Не установлены все dependencies выполните dotnet restore
Конфликт версий NET убедитесь что установлена версия SDK соответствующая проекту
Не настроена строка подключения в appsettingsjson

6.2 Ошибка подключения к PostgreSQL
Сервер PostgreSQL не запущен запустите службу PostgreSQL
Неверные данные в строке подключения хост порт имя БД логин пароль
База данных не создана выполните SQL-скрипты для создания таблиц

6.3 Ошибки при работе с приложением не загружаются данные
API недоступно проверьте запущен ли бэкенд
CORS-политика если фронтенд и бэкенд на разных портах убедитесь что в настройках бэкенда разрешены запросы с адреса фронтенда
В консоли браузера F12 могут быть видны конкретные ошибки JavaScript или ответы сервера с кодами 500
