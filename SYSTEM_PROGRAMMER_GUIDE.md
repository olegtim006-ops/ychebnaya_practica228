Руководство для системного программиста

1. Сведения о программе

Веб-приложение для учета сотрудников предприятия. Заказчик: ООО "Конди". Исполнитель: Сервисный центр "IT-Сфера".

Программный модуль реализован через:

BACKEND: ASP.NET Core
FRONTEND: HTML, CSS, JavaScript
База данных: PostgreSQL

2. Требования к программным и техническим средствам

2.1 Технические требования
Для работы программы рекомендуется использовать персональный компьютер или сервер со следующими характеристиками:

процессор: 1-2 ядра;
оперативная память: не менее 1 ГБ;
свободное место на диске: не менее 10 ГБ;
доступ к локальной сети или localhost.

2.2 Программные требования
Для развертывания и запуска программы требуется следующее программное обеспечение:

операционная система: Windows 10/11 или Linux Ubuntu/Debian;
NET SDK (версия согласно проекту);
PostgreSQL 14 или выше;
pgAdmin 4 или DBeaver для администрирования БД;
современный веб-браузер: Google Chrome, Microsoft Edge, Яндекс Браузер;
Git при необходимости для работы с репозиторием.

3. Работа с приложением

3.1 База данных

SQL-скрипты для создания таблиц и заполнения данными находятся в документации к проекту листинг 2.1. Все таблицы создаются командами CREATE TABLE с необходимыми связями и индексами. Кодировка данных UTF-8.

3.2 Настройка конфигурации приложения

Для корректной работы необходимо настроить файл appsettings.json. В целях безопасности этот файл не выкладывается в публичный репозиторий.

Необходимо настроить: DefaultConnection - подключение к БД PostgreSQL.

3.3 Первоначальный запуск

Убедитесь, что все необходимые NuGet пакеты установлены:

dotnet add package Microsoft.EntityFrameworkCore
dotnet add package Npgsql.EntityFrameworkCore.PostgreSQL
dotnet add package Microsoft.EntityFrameworkCore.Design

В каталоге проекта выполнить:

dotnet restore
dotnet build
dotnet run

После запуска в консоли появится адрес по которому будет доступен сайт обычно https://localhost:5001 или http://localhost:5000.

3.4 Проверка запуска

После старта необходимо убедиться что:

сервер запускается без ошибок;
главная страница приложения открывается в браузере;
поиск и фильтрация сотрудников работают;
операции добавления редактирования и удаления выполняются корректно;
админ-панель с журналами действий и входов доступна.

4. Структура проекта backend

Проект выполнен на базе ASP.NET Core:

Program.cs - запуск программы и настройка сервисов;

Models/ - модели данных соответствующие таблицам базы данных Employees, Departments, Positions и другие;

Data/ - контекст базы данных где объявляются таблицы и связи между ними;

Controllers/ - обработчики запросов с базовой CRUD структурой;

appsettings.json - настройки приложения включая подключение к PostgreSQL.

5. Структура проекта frontend

HTML-страницы - основная страница приложения с таблицей сотрудников и фильтрами;

site.css или встроенные стили - оформление интерфейса;

JavaScript код - клиентская логика асинхронные запросы к API рендеринг таблиц обработка действий пользователя.

6. Возможные ошибки

6.1 Приложение не запускается
Не установлены все зависимости через dotnet restore;
Конфликт версий NET SDK;
Ошибка в подключении БД.

6.2 Ошибка подключения к PostgreSQL
Сервер PostgreSQL не запущен;
Неверные данные подключения в appsettings.json;
База данных не создана.

6.3 Ошибки при работе с приложением
API недоступно - проверьте запущен ли бэкенд;
Проблемы с CORS если фронтенд и бэкенд на разных портах;
В консоли браузера F12 можно увидеть конкретные ошибки.
